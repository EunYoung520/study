# 🌸 Spring Pastel Tetris (C++ Class Version)

## 📌 프로젝트 소개

이 프로젝트는 기존의 C 기반 테트리스 게임을 **C++ 클래스 구조로 재구성**하고,
raylib 라이브러리를 사용하여 구현한 게임이다.

기본적인 테트리스 기능을 유지하면서,
파스텔톤 색상을 적용하여 **봄 느낌의 UI**를 추가하였다.

---

## 🎮 실행 화면 특징

* 🤍 배경: 흰색
* 🌸 벽: 연한 분홍색
* 💗 움직이는 블록: 블록 종류별 **진한 색상**
* 🌈 쌓인 블록: 블록 종류별 **파스텔 색상**
* 🎯 블록 모양(I, O, T, L, S)에 따라 색상이 달라 구분 가능

---

## ⚙️ 사용 기술

### 🔹 C++

* 클래스 기반 객체지향 프로그래밍
* enum class 사용
* 배열 및 구조적 데이터 관리

### 🔹 raylib

* 2D 그래픽 출력
* 키보드 입력 처리
* 게임 루프 구현

---

## 🧱 클래스 구조

### 1️⃣ Tetromino 클래스

* 블록의 모양과 타입을 관리
* 랜덤 블록 생성
* 블록 회전 기능 구현

```cpp
class Tetromino {
    array<array<int,4>,4> shape;
    BlockType type;
};
```

---

### 2️⃣ Game 클래스

* 전체 게임 상태 관리
* 블록 이동, 충돌 검사, 줄 삭제
* 화면 출력 및 게임 루프 처리

```cpp
class Game {
    GridSquare grid[][];
    Color colorGrid[][];
};
```

## ⭐ C → C++ 변경 핵심 포인트

기존 C 코드에서는
👉 전역 변수 + 함수 중심 구조였다면

본 프로젝트에서는
👉 **클래스를 활용한 객체지향 구조**로 변경하였다.

### ✔ 주요 변경 사항

* 전역 변수 제거 → 클래스 내부 변수로 캡슐화
* 함수 분리 → 멤버 함수로 구조화
* 블록 / 게임 로직 분리
* 상태 관리 명확화

## 💬 느낀 점

C 코드로 작성된 프로그램을 C++ 클래스 구조로 변환하면서
객체지향 프로그래밍의 장점을 직접 체험할 수 있었다.

특히 블록과 게임 로직을 클래스로 분리하면서
코드의 가독성과 유지보수성이 크게 향상되었다.

또한 raylib을 사용하여 그래픽을 구현하면서
단순한 콘솔 프로그램이 아닌 실제 게임 형태로 확장할 수 있었다.
